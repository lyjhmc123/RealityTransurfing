# 비주얼 디렉션 & UX 개선 전략

Intertext Magazine Issue No.1 — Reality Transurfing

---

## 1. 키 비주얼 디렉션

### 핵심 콘셉트: "변이 공간으로의 하강"

트랜서핑의 세계관은 "보이지 않는 정보장"에 대한 이야기다.
비주얼 언어는 **보이지 않는 것을 감각하게 만드는** 방향으로 설계한다.

---

### 컬러 전략

| 요소 | 값 | 역할 |
|------|-----|------|
| Primary Surface | `#12100E` (Warm Black) | 변이 공간 = 어둠 속의 무한한 가능성 |
| Light Surface | `#F5F2EE` (Wall Tint White) | 현실 세계 = 물질화된 밝은 표면 |
| Accent | `#FFC66E` (3800K Accent) | 의식의 방향 / 에너지 포인트 |
| Particle Base | `rgba(245, 242, 238, 0.15~0.6)` | 정보장의 입자들 |

#### Accent 사용 원칙

`#FFC66E`는 **"의식이 향하는 곳"** 에만 사용한다.

- CTA 버튼
- 활성 인디케이터
- 에너지가 집중되는 파티클의 하이라이트
- Survey 결과 수치

그 외에는 사용하지 않는다. 극도의 절제가 이 색상에 의미를 부여한다.

---

### 기하학적 모티프

Unsplash 사진 대신, 추상적 기하학 비주얼로 키 비주얼을 구성한다.

| 모티프 | 형태 | 의미 연결 |
|--------|------|----------|
| **무한 격자 (Infinite Grid)** | 원근감 있는 점 격자가 사방으로 뻗어나감 | 변이 공간의 무한한 섹터 |
| **동심원 파동 (Concentric Ripple)** | 중심에서 퍼져나가는 원형 파문 | 의도가 현실로 퍼져나가는 과정 |
| **진동 곡선 (Oscillation)** | 사인파가 교차하며 간섭 패턴 형성 | 진자의 에너지 흡수 메커니즘 |
| **중력장 왜곡 (Field Distortion)** | 평평한 격자가 특정 지점에서 휘어짐 | 중요도/과잉 잠재력에 의한 공간 왜곡 |
| **흐름선 (Flow Lines)** | 물처럼 자연스럽게 흐르는 파티클 경로 | 외부 의도 = 억지로 밀지 않고 허용 |

---

### 타이포그래피 방향

**Cormorant Garamond** (헤딩) + Pretendard (본문) 조합.
클래식 세리프의 권위와 신비로움이 트랜서핑의 세계관을 담는다.

키 비주얼 타이틀에 **극단적 사이즈 대비** 적용:
- Intro "Reality Transurfing" → 화면을 가득 채우는 초대형 (h1: 3.5rem, 히어로: 4.5rem)
- body text → 극도로 절제된 작은 사이즈, 넉넉한 행간 (lineHeight: 1.8)
- overline/button → Cormorant Garamond uppercase, 넓은 letterSpacing으로 장엄한 라벨

이 대비가 "무한한 공간 속 작은 존재"라는 트랜서핑의 세계관을 반영한다.

## 2. 스토리텔링 & 인터랙션 개선점

### 현재 문제 진단

| 문제 | 현재 상태 | 영향 |
|------|----------|------|
| **패턴 반복** | Article/Story가 동일한 풀스크린+인디케이터 구조 | 6-7번 섹션에서 시각적 피로 |
| **마퀴 중복** | Comments/Survey 모두 MarqueeContainer | 동일 인터랙션이 2회 반복 |
| **전환의 부재** | 라이트→다크가 즉시 전환 | "입장" 메타포가 체감되지 않음 |
| **Depth의 장식성** | PerspectiveTransition이 Terms 카드에만 적용 | 3D 효과가 스토리와 무관한 장식 |
| **단절된 섹션** | 각 섹션이 독립된 100vh 블록 | "전시장을 걸어다니는" 연속성 부족 |
| **정적 배경** | 모든 다크 섹션이 동일한 `#12100E` 단색 | 공간의 깊이감 없음 |

---

### 개선 방향: "점진적 하강" 내러티브

전시장 메타포를 "수평 이동"이 아닌 **"수직 하강"** 으로 재해석한다.
독자는 현실(라이트)에서 변이 공간(다크)으로 점점 깊이 내려가고, 다시 올라온다.

```
Intro (표면)          ← 밝고 넓은 공간, 파티클 거의 없음
  ↓ 스크롤: 배경이 점진적으로 어두워짐
Editor's Letter       ← 어둠 속으로 입장, 파티클 서서히 등장
  ↓
Index                 ← 길 안내 (전시장 로비)
  ↓
Comments              ← 다른 관람객의 목소리 (수평 흐름)
  ↓
Terms                 ← 핵심 전시실 — 각 용어별 인터랙티브 비주얼
  ↓
Article               ← 깊이 있는 감상 (텍스트 중심, 비주얼은 배경)
  ↓
Story                 ← 개인적 체험담 (Article과 시각적 차별화)
  ↓
Survey                ← 데이터 시각화 (마퀴 대신)
  ↓ 스크롤: 배경이 점진적으로 밝아짐
Outro                 ← 변이 공간에서 현실로 복귀
  ↓
Footer (표면)          ← 다시 밝은 세계
```

---

### 섹션별 개선안

| 섹션 | 현재 | 개선안 |
|------|------|--------|
| **Intro → Editor** | 즉시 다크 전환 | GradientOverlay 활용. 스크롤에 따라 배경이 **점진적으로** `#F5F2EE` → `#12100E` 전환. 파티클이 서서히 나타남 |
| **Comments** | 마퀴 카드 수평 흐름 | **유지**. "웅성거림"의 시각화로 적절함 |
| **Terms** | PerspectiveTransition + 가로 스크롤 | 가로 스크롤 유지. 각 카드에 **Three.js 파티클 비주얼**을 배경으로 적용. PerspectiveTransition 제거하고 **스크롤 진입 시 파티클이 형태를 잡는** 애니메이션으로 대체 |
| **Article** | 풀스크린 사진 + 오버레이 | 사진 제거. **기하학적 배경 패턴**(각 아티클 주제에 맞는)으로 대체. 텍스트가 주인공 |
| **Story** | Article과 동일 구조 | **좌우 분할 레이아웃**으로 차별화. 좌측에 인터뷰이 실루엣/추상 이미지, 우측에 텍스트. SplitScreen 컴포넌트 활용 |
| **Survey** | MarqueeContainer (Comments와 동일) | **스크롤 연동 데이터 시각화**로 변경. 숫자가 0에서 카운트업, 원형/바 그래프 파티클이 채워지는 애니메이션 |
| **Outro → Footer** | 즉시 라이트 전환 | 파티클이 서서히 사라지며 다크→라이트 점진 전환 |

---

### Depth(z-depth) 활용 판단

현재 PerspectiveTransition의 "뒤로 누워있다가 세워지는" 효과는 **스토리와 연결되지 않아 장식적**이다.

#### Depth가 의미를 갖는 경우

| 기법 | 적용 | 효과 |
|------|------|------|
| **패럴랙스** | 전체 섹션 배경 | 스크롤 시 배경 파티클은 느리게, 텍스트는 빠르게 → "공간 속을 이동하는" 감각 |
| **줌 전환** | Terms 카드 → 상세 모달 | 클릭하면 해당 비주얼로 "빨려들어가는" 전환 |
| **레이어링** | 파티클 배경 → 기하학 모티프 → 텍스트 | 3개 레이어가 다른 속도로 움직여 공간감 형성 |

#### Depth가 적합하지 않은 경우

| 기법 | 이유 |
|------|------|
| 카드 3D 회전 등장 (현재 PerspectiveTransition) | 형식적. 모든 카드에 동일 적용되면 의미 없음 |
| 슬라이드 전환에 z-axis 이동 | 수평 콘텐츠 탐색에 깊이감은 혼란 유발 |

**결론**: Depth는 **"전체 공간"에 적용**(패럴랙스, 레이어 분리)하되, **개별 요소 단위**(카드 회전, 슬라이드 depth)에는 적용하지 않는다.

## 3. 인터랙티브 비주얼 전략: 개념별 Three.js 시각화

### 설계 원칙

- 각 Terms 용어에 **고유한 파티클/기하학 비주얼 1개씩** 매핑
- 비주얼은 **개념을 설명하는 도구**이지 장식이 아님
- 스크롤 또는 마우스/터치로 상호작용 가능
- 모두 모노크롬(`#F5F2EE` 파티클 on `#12100E` 배경) + `#FFC66E` 포인트

---

### 변이 공간 (Variants Space) — Infinite Particle Grid

```
형태:     3D 공간에 균일하게 배치된 수천 개의 점 격자.
          원근법으로 무한히 뻗어나가는 느낌.

상호작용:  마우스 위치에 따라 카메라 시점 미세 이동 (패럴랙스).
          스크롤 시 격자 사이를 관통하며 이동하는 느낌.

의미:     "모든 가능한 현실이 이미 존재한다."
          각 점이 하나의 가능한 현실, 무한히 펼쳐진 가능성.

핵심 요소: 점 하나가 #FFC66E로 빛남 = "당신이 선택한 현실"
```

---

### 진자 (Pendulum) — Oscillating Attractor

```
형태:     중앙에 큰 구체(진자), 주변에 떠도는 작은 파티클(개인).
          진자가 좌우로 진동하며 파티클을 끌어당김.

상호작용:  마우스를 진자 쪽으로 가져가면 파티클이 빨려들어감.
          마우스를 멀리 옮기면 파티클이 자유로워짐.

의미:     "반응하면 할수록 에너지를 빼앗긴다."
          가까이 갈수록 흡인력 강해지는 것을 직접 체감.

핵심 요소: 진자에 흡수된 파티클은 어두워지고, 자유로운 파티클은 밝음.
```

---

### 중요도 (Importance) — Gravity Well

```
형태:     평평한 2D 격자(와이어프레임 메쉬).
          특정 지점에 "무게"가 가해져 격자가 움푹 들어감.

상호작용:  클릭/터치한 지점에 중력이 생겨 격자가 왜곡됨.
          길게 누르면 왜곡이 심해지고 주변 파티클이 빨려들어감.
          놓으면 서서히 평평하게 복원.

의미:     "과도한 의미 부여가 공간을 왜곡시킨다."
          집착하면 공간이 찌그러지고, 놓아주면 원래대로.

핵심 요소: 왜곡이 심할수록 #FFC66E 강도 증가 → 과잉 잠재력의 위험 신호.
```

---

### 과잉 잠재력 (Excess Potential) — Repulsion Field

```
형태:     중앙에 에너지가 축적되는 구체.
          파티클이 중앙으로 수렴하다가 임계점에서 폭발적으로 반발.

상호작용:  스크롤 진행에 따라 자동 재생.
          축적 → 임계 → 반발 → 안정의 사이클 반복.

의미:     "중요도가 높아지면 균형 세력이 반대 방향으로 작용한다."
          쌓으면 쌓을수록 터지는 것을 시각적으로 체감.

핵심 요소: 반발 순간 파티클 색이 #FFC66E → 흰색으로 플래시.
```

---

### 외부 의도 (Outer Intention) — Flow Field

```
형태:     Perlin noise 기반 유동장(flow field).
          파티클이 자연스러운 곡선을 따라 흐름.
          억지로 직선 이동하는 파티클(내부 의도)과
          유동장을 따라 곡선으로 흐르는 파티클(외부 의도) 대비.

상호작용:  마우스로 흐름을 따라가면(허용) 파티클이 합류.
          마우스로 억지로 밀면(저항) 파티클이 흩어짐.

의미:     "밀어붙이지 않고 허용하면 변이 공간이 길을 연다."
          자연스러운 흐름 vs 억지 직선의 차이를 느낌.

핵심 요소: 흐름에 합류한 파티클만 목적지에 도달하며 #FFC66E로 빛남.
```

---

### 구현 전략

| 구분 | 접근 |
|------|------|
| 렌더러 | Three.js (이미 `GradientOverlay`에서 사용 중) |
| 파티클 시스템 | `THREE.Points` + `BufferGeometry` + custom shader |
| 성능 | GPU 인스턴싱. 파티클 수: 모바일 2,000개 / 데스크탑 10,000개 |
| 스크롤 연동 | Framer Motion `useScroll` → Three.js uniform으로 전달 |
| 컴포넌트 위치 | `src/components/dynamic-color/` 하위에 개념별 컴포넌트 |
| fallback | WebGL 미지원 시 CSS gradient + SVG 정적 이미지 |

---

### 적용 위치

| 비주얼 | 적용 위치 |
|--------|----------|
| Infinite Particle Grid | Intro 배경 + Terms "변이 공간" 카드 |
| Oscillating Attractor | Terms "진자" 카드 |
| Gravity Well | Terms "중요도" 카드 + "과잉 잠재력" 카드 (강도 차이로 공유) |
| Repulsion Field | Article 섹션 배경 (미묘하게) |
| Flow Field | Outro 배경 (마무리: 모든 것이 자연스럽게 흐르는 상태) |
